# RFC Process

## When RFC Is Required
- Repo topology changes
- Cross-subproject dependency changes
- Release/versioning strategy changes
- New external platform/tool integrations

## RFC Template
1. Context
2. Problem statement
3. Proposed change
4. Alternatives considered
5. Risks + mitigations
6. Rollout plan
7. Rollback plan
8. Owner + approvers

## Lifecycle
- `Draft` -> `Review` -> `Accepted` -> `Implemented` -> `Archived`
