# Contribution Guide

## Branch + PR Rules
- Branch from `main` using `feat/*`, `fix/*`, `ops/*`, or `docs/*`.
- Every PR requires CODEOWNERS approval.
- CI must pass before merge.
- Direct push to `main` is not allowed.

## PR Checklist
- [ ] Problem statement and scope are clear.
- [ ] Change is limited to intended subproject(s).
- [ ] Risk notes included.
- [ ] Rollback approach documented for non-trivial changes.
- [ ] Tests/manual validation evidence included.

## Definition of Done
- Governance checks pass.
- Security scan passes.
- Docs updated for behavior/config changes.
